\section{Passo a Passo}

	O algortimo segue os passos abaixo:
		\begin{itemize}
			\item Definir as (regiões de interesse)
			\item Considera a imagem como um grafo
				\begin{itemize}
					\item Nós do grafo = pixels
					\item Arestas = conectam os pixels usando distâncias ponderadas
				\end{itemize}
			\item Calcula a distância geodésica (custo das arestas) utilizando \ref{eq:distgeodesica} e \ref{distpixel}
			\begin{itemize}
	 \item Cálculo dos pesos
	 	\begin{itemize}
	 		\item Definir um conjunto de imagens $F_{i}$ e escolher um subconjunto a partir dessas imagens
	 		\item Definir pesos diferentes para cada canal $N_{c}$(19 canais foram utilizados). \textcolor{red}{Selecionar o número de canais é um ponto crítico pois um grande número é necessário para atender a um maior espectro de imagens, porém as informações mais importantes de uma dada imagem normalmente se concentram em poucos canais}
	 		\item Selecionar os canais usando 16 filtros gabor mais 2 canais de crominância ($C_{b}$ e $C_{r})$ e um de luminância. \textcolor{red}{O framework foi projetado para ser utilizado com qualquer tipo de filtro}
	 			\begin{itemize}
	 				\item A saída dos filtros de Gabor são normalizadas devido ao ruído e complexidade presentes em imagens naturais. 
	 			\end{itemize}
	 		\item Calcular pesos diferentes para cada canal para da imagem. (\textcolor{red}{robustez do algoritmos, se adapta aos dados de entrada})
	 		\item Dado um conjunto de pixels ($\Omega_{1}, \Omega_{2} $) correspondentes aos ``rabiscos'' $l_{1}, l_{2} $, a função densidade de probabilidade (FDP) é aproximada para uma gaussianda com $\Omega_{1},\Omega_{2} $.
	 		\item Calcula-se a probabilidade de um pixel \textit{x} pertencer ao ``rabisco'' $l_{1}$ baseado no canal $F_{i}$
	 		\item A probabilidade de um pixel \textit{x} pertencer a uma região $l_{i}$ é dada pelo somatório da probabilidade do mesmo pixel \textit{x} calculada para cada canal multiplicada pelo peso do mesmo canal.
	 		\item Para a separação de múltiplipas regiões o cálculo é parecido, a diferença está no cálculo da função peso $W_{i}$, que é o somatório da função peso da região atual competindo com todas as outras regiões (essa comparação ocorre sempre aos pares) ($W_{ij}$)
	 		\item O algoritmo assume que o usuário irá sempre marcar regiões da imagem que são diferentes e a informação contida nessas regiões poderá ser usada para separar as diferentes texturas.		
	 		
	 	\end{itemize}
	\end{itemize}
			\item Baseado no cálculo anterior, encontra-se a probabilidade de um determinado pixel pertencer a um dada região de interesse usando \ref{probabilidade}
		\end{itemize}
		
		\begin{equation}
			d(s,t) := min_{C_{s,t}} \int |\nabla Y \dot{C}_{s,t}(p)| dp
			\label{eq:distgeodesica}		
		\end{equation}
		
		\begin{equation}
			d_{i}(t) = min_{s \in \Omega _{c}:label(s)=l_{i}}d(s,t)
			\label{distpixel}
		\end{equation}
		\begin{equation}
			Pr(t \in l_{i}) = \frac{d_{i}(t)^{-1}}{\sum _{j \in[1,N_{l}]} d_{j}(t)^{-1}} 
			\label{probabilidade}  
 		\end{equation}

As técnicas de segmentação citadas no texto são em sua maioria para imagens coloridas ou em níveis de cinza, onde o gradiente da intensidade (ou da cor) é baixo na região de interesse e alto nas bordas, entretanto esta afirmação não é válida para imagens mais complexas (com texturas, por exemplo). Para alcaçar segmentação nessas imagens o autor usa um conjunto adaptativo de filtros Gabor e escolhe a função peso da distância geodésica de acordo com a imagem.

A complexidade do algoritmo é linear em relação ao número de pixels graças ao uso da distância geodésica para o cálculo da probailidade de um pixel pertencer à uma região.

A abordagem é interativa onde o objetivo principal é adicionar cor ( ou outro efeito) à uma imagem, no artigo tomado como base pelo autor são dados uma série de ``rabiscos'' (que marcam regiões de interesse na imagem) em uma imagem somente de luminância, e então é calculada a distância geodésica do mesmo canal de luminância para posteriormente calcular a probabilidade de um pixel pertencer a um dado ``rabisco'' . \subsection{Anotações sobre os pesos}


Cada ``rabisco'' possui uma função peso. As distâncias ponderadas devem ser calculadas para cada ``rabisco'', entretanto não precisam, necessariamente ser calculadas para toda a imagem (i.e, se um pixel já recebeu um valor baixo o suficiente pode-se assumir que esse pixel já pertence à uma dada região da imagem).


Os canais são escolhidos baseados no conjunto de pixels $\Omega _{c}$ definido pelo usuário, assumido que o usuário irá sempre marcar  com \textit{labels} diferentes regiões que sejam de fato diferentes na imagem.
